## UI Development Steps

### Phase 1: Core Components (Week 1)
**Goal**: Build reusable agenda components

1. **AgendaItemCard** - Display agenda item with all details
2. **AgendaItemForm** - Create/edit agenda item modal
3. **AgendaItemsList** - List with drag-and-drop support
4. **AgendaStatusBadge** - Status indicator (draft/published/archived)
5. **ItemTypeTag** - Item type indicator with icon
6. **TimeAllocationDisplay** - Show estimated vs actual time

### Phase 2: Agenda Editor Page (Week 1-2)
**Goal**: Create/edit agenda workflow

7. **AgendaEditorPage** - Main editor layout
8. **AgendaHeader** - Meeting info, status, actions
9. **TemplateSelector** - Choose template modal
10. **PublishAgendaModal** - Pre-publish checklist

### Phase 3: Agenda Viewer (Week 2)
**Goal**: View published agenda

11. **AgendaViewPage** - Read-only agenda view
12. **AgendaItemDetails** - Expanded item view with documents
13. **AgendaPdfViewer** - Display generated PDF

### Phase 4: Meeting Execution (Week 3)
**Goal**: Execute agenda during meeting

14. **AgendaExecutionPanel** - Real-time agenda in meeting room
15. **CurrentItemHighlight** - Highlight active item
16. **AgendaTimer** - Track time per item
17. **AddAdHocItemButton** - Quick add during meeting

### Phase 5: Templates (Week 3)
**Goal**: Manage agenda templates

18. **AgendaTemplatesPage** - List all templates
19. **TemplateCard** - Template preview card
20. **TemplateEditorPage** - Create/edit template

**Logic**:
```typescript
const renderAgendaTab = () => {
  const { data: agenda, isLoading } = useAgenda(meeting.id);

  if (isLoading) return <Spin />;

  if (!agenda) {
    return <EmptyAgendaState onCreateAgenda={handleCreateAgenda} />;
  }

  if (agenda.status === 'draft' && canEdit) {
    return <AgendaEditor agendaId={agenda.id} meetingId={meeting.id} />;
  }

  return <AgendaViewer agenda={agenda} canEdit={canEdit} />;
};
```
